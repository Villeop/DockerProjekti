Docker yleisesti

Docker on avoimen lähdekoodin alusta, jonka avulla on mahdollista pakata sovelluksia ja niiden riippuvuuksia kevyisiin kontteihin. Docker kontit ovat kuin eräänlaisia virtuaalisia ympäristöjä, jotka ovat riippumattomia käyttöjärjestelmästä ja alustasta. Tämä mahdollistaa ohjelmistojen siirtämisen paikasta toiseen ilman, että ohjelmistoa tarvitsee muokata jokaista alustaa varten. Koska kontit ovat helposti siirrettävissä ja skaalattavissa, tehostavat ne sovellusten kehitystä, testausta ja julkaisua.

Miksi kontteja tarvitaan ja miten ne eroavat virtuaalipalvelimista?

Jokainen luotu ohjelma on riippuvainen käyttöjärjestelmästä, erilaisista kirjastoista, lisäosista ja asetuksista. Tämä voi aiheuttaa ongelmia, jos ohjelmaa halutaan ajaa eri alustoilla. Yksi ratkaisu tähän on yleensä virtuaalipalvelimet, jotka tarjoavat ohjelmille niiden tarvitseman ympäristön. Virtuaalipalvelimien heikkous on, että ne ovat raskaita ja vievät paljon resursseja, sillä suurin osa virtuaalipalvelimien sisällöstä on merkityksetöntä.

Kontit taas ovat kevyempiä ja nopeampia, koska ne jakavat käyttöjärjestelmän ytimen ja resurssit isäntäkoneen kanssa. Tämä mahdollistaa useiden konttien ajamisen samalla isäntäkoneella, sovellusten nopeamman käynnistyksen, sekä siirron ympäristöjen välillä ja jopa 100 kertaa kooltaan pienemmän kontti-imagen verrattuna virtuaalipalvelimeen.

Dockerin peruskomponentit

Docker koostuu useista komponenteista, joita ovat:

  • Docker Image: Docker Image on tiedosto, joka tarjoaa ohjeet kontin luomiseen. Se sisältää itse sovelluksen, sekä sen tarvitsemat ohjelmistot ja käyttöjärjestelmän.
  • Docker-kontti: Kontti on imagen pohjalta käynnistetty sovellusyksikkö. Kontit ovat kevyitä ja helposti siirrettäviä paketteja, jotka sisältävät sovelluksen, sekä sillä tärkeät riippuvuudet
  • Dockerfile: Dockerfile sisältää ohjeet Docker Imagen rakentamiselle. Se sisältää komennot mm. riippuvuuksien asentamiseen, tiedostojen kopiomiseen, sekä ympäristön määrittämiseen.
  • Docker Engine: Docker Engine on ohjelmisto, joka mahdollistaa konttien luomisen, hallinnoinnin ja ajamisen. Se koostuu Docker Daemonista, REST API:sta ja Docker CLI:stä. Tätä voidaan hallita Docker APIn kautta esim. PowerShellilla tai Bashilla.
  • Docker Hub: Docker Hub on pilvipalvelu, joka tarjoaa julkisia Docker Imageja. Se mahdollistaa Imagen jakamisen ja löytämisen, sekä yhteistyön muiden sovelluskehittäjien kanssa.

Yhteenvetona

Docker kontit siis nopeuttavat, sekä yksinkertaistavat ohjelmistokehitystä pitämällä kehitysympäristöt samannäköisinä ja tiedostokoot pieninä. Verrattuna perinteiseen palvelimiin perustuvaan malliin, on Dockerin kontti pohjainen malli tehokkaampi ja skaalautuvampi, joka myös säästää resursseja.
